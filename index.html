UNIT I - Introduction to C++

1. Historical Background and Detailed Evolution

C++ is a multi-paradigm, statically typed, compiled, general-purpose programming language conceptualized and developed by the Danish computer scientist Bjarne Stroustrup in the year 1979 at AT&T Bell Laboratories located in Murray Hill, New Jersey. Initially, the project was started to address severe software development limitations in massive network simulations. Stroustrup combined the execution speed and low-level control of the C language with the high-level data abstraction capabilities found in Simula67.

The language was originally titled "C with Classes". However, by 1983, as more capabilities like operator overloading, references, and virtual functions were integrated, the language was permanently renamed C++. The symbol "++" represents the C increment operator, explicitly indicating that the language is an advanced extension of C. It achieved official ISO standardization in 1998 (C++98) and has undergone modern transformations through updates like C++11, C++14, C++17, C++20, and C++23.

2. Core Architectural Features of C++

  • Middle-Level Language Design: C++ bridges low-level architectural control (direct memory mapping, system pointers) and high-level abstract logic (classes, interfaces), providing optimal flexibility.
  • Compiled and Fast Execution: Code is directly compiled into hardware-specific native machine instructions bypassing any virtual runtime interpreter layers, making it significantly faster than Java or Python.
  • Explicit Memory Management Control: Programmers are given total authority to allocate and deallocate heap memory memory slots at runtime using the new and delete keywords, optimizing resource management.
  • Rich Standard Template Library (STL): It includes a massive collection of predefined data structures and complex algorithms like vectors, lists, queues, maps, sorting, and searching operations out of the box.

UNIT I - Key Concepts of Object-Oriented Programming (OOP)

1. Paradigm Shift & Foundational Philosophy

Traditional structural programming models focus entirely on algorithmic procedures, treating data as an secondary component that moves freely throughout global environments. Object-Oriented Programming (OOP) reverses this architecture by shifting primary focus onto Data Security. It structures software systems around modular, self-contained components called Objects, which combine data and related operations into a single package.

2. Technical Architecture of the Four Essential OOP Pillars

  1. Encapsulation (Data Binding): The process of binding data members and data processing methods into a single structural block called a Class. This prevents external routines from accessing internal attributes directly.
  2. Data Abstraction (Interface Hiding): Displays only the essential operational features to the outside user while hiding complex structural backend logic. This is implemented using access specifiers like public, private, and protected.
  3. Inheritance (Code Reuse): The mechanism that allows a new class (derived subclass) to inherit data parameters and functional methodologies from an existing class (base parent class), reducing code duplication.
  4. Polymorphism (Multi-Form Processing): Allows a single interface name or operator identifier to execute different tasks depending on context. It is split into compile-time (overloading) and runtime (virtual functions) stages.

UNIT I - Advantages of Object-Oriented Programming (OOP)

1. Exhaustive Software Engineering Advantages

Adopting an Object-Oriented architectural design patterns offers key long-term benefits for modern software engineering teams:

  • Enhanced System Modularity: Because every class operates as an independent module with its own scope, updates or changes can be made inside a class without affecting other parts of the application.
  • Effective Code Reusability: Inheritance allows developers to reuse tested, proven source code from base classes, which accelerates development and reduces testing overhead.
  • Streamlined Maintenance and Debugging: Isolating data errors to localized object instances makes debugging faster, as engineers can isolate and fix issues without reading through the entire codebase.
  • Robust Security Models: Data hiding via private encapsulation safeguards critical internal application variables from accidental overwrites or malicious external access.

UNIT I - Object-Oriented Languages

1. Typology and Classification: Pure vs. Hybrid Engines

Object-oriented languages are classified into two groups based on how strictly they enforce OOP design rules: Pure Object-Oriented Languages and Hybrid Object-Oriented Languages.

In a pure OOP language (e.g., Smalltalk, Ruby), every variable and data element must be represented as an object. Primitive types (like native integers or characters) cannot exist outside an object context. Hybrid languages (e.g., C++, Java, Python) allow both structured procedural workflows and object-oriented programming to be used within the same application.

UNIT I - Iterative Frameworks (Loops in C++)

1. In-Depth Operational Taxonomy of Loop Categories

Loops run a target block of code repeatedly as long as a specified conditional expression remains true. When the condition evaluates to false, loop execution terminates, and control returns to the main program flow. C++ categorizes loops into two functional divisions based on where the condition is evaluated:

  • Entry-Controlled Loops: The condition expression is evaluated at the start of the loop cycle, before any internal statements execute. If the condition is false on the first check, the loop body is skipped entirely. Examples include the for loop and the while loop.
  • Exit-Controlled Loops: The condition expression is checked at the bottom of the loop cycle, after the body executes. This structure guarantees that the loop statements run at least once, regardless of whether the condition is true or false. The primary example is the do-while loop.

2. The 'for' Loop Architecture (Fixed Iterations Optimization)

The for loop is an entry-controlled loop structure used when the exact number of iterations is known before execution. It consolidates the counter initialization, condition evaluation, and counter increment/decrement expressions into a single control line, making the code clean and highly readable.

UNIT I - Optimization Frameworks (Inline Functions)

1. Function Call Overhead Optimization Strategy

In standard execution models, executing a function call incurs a performance cost known as Function Call Overhead (saving register states, copying arguments to stack frames, and performing jump jumps). For small, frequently called functions, this overhead can degrade performance.

C++ addresses this issue using the inline keyword. This instructs the compiler to replace the function call site directly with the function's raw body logic during compilation, removing the execution overhead of a standard call.

UNIT III - Overloading Unary and Binary Operators

1. Definition

Binary Operator Overloading: Rendu operands oda velai seiyra operators ah overload panradhu. Ex: +, -, *, /, ==. Member function ah ezhudina oru argument podhum. Friend function ah ezhudina rendu arguments venum.

Explanation - Tamil la:

1. Basic to Class: int, float maari basic type ah class object ah maathuradhu. Idhukku Constructor use aagum.
2. Class to Basic: Class object ah int, float maari basic type ah maathuradhu. Idhukku Casting Operator Function use aagum.
3. Class to Class: Oru class oda object ah innoru class oda object ah maathuradhu. Idhukku Constructor illa Casting Operator use pannalam.

Explanation - Tamil la:
1. Basic to Class: ATM la 1000 ruba notes podura maari. Machine ku ulla adhu oru object aagidum. int x = 5; ah Time t = x; nu maarina maari. Constructor call aagum.
2. Class to Basic: Object la irukura data ah velliya eduthu basic variable la poduradhu. Time t(2,30); ah int mins = t; nu sonna total minutes ah eduthu kudukum. operator int() function call aagum.
3. Class to Class: Tamil book ah English la translate pandra maari. Rectangle r; ah Square s = r; nu maaruradhu. Destination class la constructor illa source class la operator function ezhudhanum.
4. En Use: obj1 = obj2 + 5; nu ezhuda 5 ah object ah maathitu add pannanum. illa object ah int ah maathi add pannanum.

#include <iostream>
using namespace std;

class Time {
private:
    int hours, minutes;

public:
    // 1. Basic to Class - Constructor
    Time(int m) {
        hours = m / 60;
        minutes = m % 60;
    }
    
    Time(int h, int m) {
        hours = h;
        minutes = m;
    }
    
    // 2. Class to Basic - Casting Operator
    operator int() {
        return hours * 60 + minutes;
    }
    
    void display() {
        cout << hours << " hours " << minutes << " minutes" << endl;
    }
};

int main() {
    cout << "--- Basic to Class ---" << endl;
    int duration = 130;
    Time t1 = duration;
    cout << "130 minutes = ";
    t1.display();
    
    cout << "\n--- Class to Basic ---" << endl;
    Time t2(2, 45);
    int totalMins = t2;
    cout << "2 hours 45 minutes = " << totalMins << " minutes" << endl;
    
    cout << "\n--- Direct Conversion ---" << endl;
    Time t3 = 95;
    cout << "95 minutes = ";
    t3.display();
    
    int x = Time(1, 20);
    cout << "1 hour 20 minutes = " << x << " minutes" << endl;
    
    return 0;
}

UNIT 3 - Inheritance

1. Definition

Inheritance: Oru class oda properties and functions ah innoru class ku eduthukkura process dhan Inheritance. Already irukura class ku Base Class illa Parent Class nu per. Pudhu class ku Derived Class illa Child Class nu per.

Idhoda main use: Code Reusability. Oru thadava ezhudhuna code ah thirumba thirumba use pannalam. Syntax: class Derived : access_specifier Base { };

Explanation - Tamil la:
1. Enna Idhu: Appa kitta irukura veedu, nilam ellaam paiyanukku varum la. Adhe maari Base class la irukura public/protected members ellaam Derived class ku automatic ah kidaikkum.
2. En Use: Vehicle nu oru class la speed, color irukku na, Car class ku adha thirumba ezhuda theva illa. class Car : public Vehicle nu sonna podhum. Code duplication kammi aagum.
3. Access Specifier: public, protected, private nu moonu irukku. public inheritance na base oda public/protected members derived ku public/protected ah varum. Private members eppovume kidayadhu.
4. Types: Single, Multiple, Multilevel, Hierarchical, Hybrid nu 5 types irukku.

UNIT 3 - Types of Inheritance

1. Definition

Types of Inheritance: C++ la 5 main types of inheritance irukku. Class ku class epdi connection irukku nu adha vachu decide aagum.

1. Single: Oru Base class la irundhu oru Derived class varradhu.
2. Multiple: Pala Base class la irundhu oru Derived class varradhu.
3. Multilevel: Chain maari Base -> Derived -> Derived nu pogum.
4. Hierarchical: Oru Base class la irundhu pala Derived class varradhu.
5. Hybrid: Mel irukura rendu illa adhigama types ah serthu use panradhu.

Explanation - Tamil la:
1. Single: Appa -> Paiyan maari. class B : public A. Simple.
2. Multiple: Appa + Amma -> Paiyan maari. Renduper oda qualities um varum. class C : public A, public B.
3. Multilevel: Thatha -> Appa -> Paiyan maari. class C : public B, class B : public A.
4. Hierarchical: Oru appa ku rendu paiyan maari. class B : public A, class C : public A.
5. Hybrid: Mix panni use panradhu. Idhula Diamond Problem varum. Adha solve panna virtual keyword use panrom.

UNIT 3 - Virtual Base Classes

1. Definition

Virtual Base Class: Multiple inheritance + Hierarchical inheritance serthu Hybrid inheritance ezhudum bodhu varra Diamond Problem ah solve panna use aagura concept dhan Virtual Base Class.

Diamond Problem na enna: Class A ah, Class B um Class C um inherit pannum. Apram Class D, B and C rendaiyum inherit pannum. Ippo D ku A oda members rendu copy varum. Ambiguity aagidum. Idha thadukka base class ah virtual nu declare pannanum.

Syntax: class B : virtual public A and class C : virtual public A

Explanation - Tamil la:
1. Problem: Thatha kitta irukura sothu appa vazhiya oru copy, amma vazhiya oru copy nu rendu thadava paiyanukku varudhu na confusion aagum la. Adhu dhan Diamond Problem.
2. Solution: Thatha sothu ku virtual nu sollidu. "Oru copy dhan kodukanum, yaar vazhiya vandhaalum" nu meaning. Appo paiyanukku oru copy mattum varum.
3. Enna Nadakkum: Compiler paathu, virtual base class na oru copy mattum than memory la create pannum. D class object ku A oda members oru thadava dhan varum.
4. Constructor Rule: Virtual base class oda constructor ah most derived class dhan call pannanum. Idhu romba mukkiyam.

UNIT 3- Abstract Classes

1. Definition

Abstract Class: Kuraindhadhu oru Pure Virtual Function vachirukura class ku Abstract Class nu peru. Idhuku object create panna mudiyadhu. Idhu base class ah mattum dhan use aagum.

Pure Virtual Function: virtual return_type function_name() = 0; nu declare panradhu. Function ku body kedayadhu. = 0 podanum. Derived class la idha kandippa override pannanum.

Use: Oru design illa structure ah define panna use aagum. "Idhu idhu functions irukkanum" nu sollidum. Aana epdi velai seiyanum nu solladhu.

UNIT IV - Pointers to Objects

1. Definition

Pointer to Object: Oru class oda object oda address ah hold panra pointer ku Pointer to Object nu peru. Normal data types ku pointer use panra maari, class objects ku use panrom.

Syntax: ClassName *ptr; apram ptr = &object; illa ptr = new ClassName;

Explanation - Tamil la:

3. Arrow ->: ptr.name nu ezhuda mudiyadhu. ptr->name illa (*ptr).name nu ezhudanum. Arrow easy.
4. this Pointer: Class ku ulla this->x = x; nu ezhudum bodhu, left side this->x data member. Right side x parameter. Name clash varum bodhu use aagum.

UNIT IV - Dynamic Object Binding

1. Definition

Explanation - Tamil la:

UNIT V - Sequential Read/Write Operations

1. Definition

Sequential File Access: File la data ah onnu aprom onnu nu varisaya ga read pannura illa write panradhu dhan Sequential Access. First record la irundhu last record varaikkum order ah pogum. Random ah jump panna mudiyadhu.

Write Operations:
1. << operator - Formatted output. Text maari write pannum.
2. put() - Single character write pannum.
3. write() - Binary data ah block ah write pannum.

Read Operations:
1. >> operator - Formatted input. Space varaikkum read pannum.
2. get() - Single character read pannum.
3. getline() - Oru line ah full ah read pannum.
4. read() - Binary data ah block ah read pannum.

Explanation - Tamil la:
1. Sequential na enna: Cassette tape maari. Paata 1 la irundhu 5 ku poga 2,3,4 ah skip panni poga mudiyadhu. Varisaya ga dhan poganum.
2. Write: file << "Arjun" << " << 85; nu sonna file la "Arjun 85" nu ezhudhum. Oru record mudinja adutha record ku poga endl podanum.
3. Read: file >> name >> marks; nu sonna first "Arjun" name ku pogum, aprom 85 marks ku pogum. Varisaya ga dhan padikkum.
4. Problem: 1000 records irundha 999 ah padikka venumna, munnadi 998 records ah padichu dhan poganum. Time aagum.
5. Use: Log files, CSV files, simple data storage ku use aagum. Random access thevai illana sequential podhum.

#include <fstream>
#include <string>

        cout << "Roll: " << s.roll << " | Name: " << s.name
             << " | Marks: " << s.marks << endl;
    }

    }
    charOut.close();

    ifstream charIn("char.txt");
    char ch;
    cout << "Character by character: ";
    while(charIn.get(ch)) {

    }
    charIn.close();
    cout << endl;

    Student s1 = {101, "Arjun", 85.5};

UNIT V - Binary and ASCII Files

1. Definition

ASCII File / Text File: Data ah human readable format la store panradhu. Characters ah ASCII code la save aagum. Notepad la open panni padikkalam. \n ah OS ku thagundha maari convert pannum. Default mode.

Binary File: Data ah memory la epdi irukko apdiye byte by byte save panradhu. Human readable ah irukkaadhu. Fast, space kammi. ios::binary mode use pannanum. \n convert aagaadhu.

Key Differences:
1. Storage: ASCII - 123 nu save panna '1''2''3' nu 3 bytes. Binary - int 123 na 4 bytes la direct ah store aagum.
2. Speed: Binary fast, yen na conversion illa. ASCII slow, yen na format pannanum.
3. Size: Binary la size kammi. ASCII la adhigam.
4. Portability: ASCII files endha system laiyum open pannalam. Binary files system dependent ah irukkalam.
5. Functions: ASCII ku <<, >>, getline(). Binary ku write(), read().

Explanation - Tamil la:
1. ASCII File: Tamil la ezhudura maari. Nee padikka mudiyum, computer kum puriyum. Aana idam adhigam aagum. "100" nu ezhuduna 3 ezhuthu.
2. Binary File: Morse code maari. Manushanuku puriyadhu, aana machine ku fast ah puriyum. "100" nu number ah 4 bytes la adakidum.
3. Eppo ASCII: Config files, CSV, logs, user edhu padikkanumo adhu..txt,.csv,.xml files.
4. Eppo Binary: Images, Videos, Audio,.exe, Database files. Size kammi venum, speed venum na. Objects ah save panna.
5. Example: int x = 12345; ASCII la "12345" nu 5 bytes. Binary la 4 bytes dhan. 1 crore numbers save panna Binary la neraya idam meedhi aagum.

UNIT V - Random Access Operation

1. Definition

Random Access: File la edhuna position ku direct ah jump panni read illa write panradhu dhan Random Access. Sequential maari varisaya ga poga thevai illa. Database records ku idhu romba use aagum.

File Pointer Functions:
1. seekg() - Get Pointer ah move pannum. Read ku use aagum. seekg(offset, direction)
2. seekp() - Put Pointer ah move pannum. Write ku use aagum. seekp(offset, direction)
3. tellg() - Get Pointer indha position la irukku nu sollum.
4. tellp() - Put Pointer indha position la irukku nu sollum.

Direction Flags:
ios::beg - Beginning la irundhu
ios::cur - Current position la irundhu
ios::end - End la irundhu

Important: Random access ku ios::binary mode use panradhu nalladhu. Yen na text mode la \n convert aagum, position kanaku thappagum.

Explanation - Tamil la:
1. Enna Idhu: CD la 5th paata kekkanum na direct ah jump pannalam. Cassette maari ellathayum fast forward panna thevai illa. Adhu dhan Random Access.
2. seekg: Book la 50th page ku poga page number ah thirupura maari. seekg(50, ios::beg) na 50th byte ku pogum.
3. tellg: Nee ippo endha page la irukka nu kekura maari. Current byte position sollum.
4. Use: Student record la 100th student data venum na, seekg(99 * sizeof(Student), ios::beg) nu direct ah pogalam. Full file ah padikka thevai illa.
5. Condition: Fixed size records irundha dhan easy. struct Student {int id; char name[20];}; maari size fix ah irukkanum.

UNIT V - Templates

1. Definition

Templates: Generic programming ku use aagura C++ feature. Orey function illa class ah different data types ku work aaga vachukalam. Compile time la compiler namma kodutha type ku thagundha maari code generate pannidum.

2 Types:
1. Function Templates: Generic functions create panna. template <typename T> T add(T a, T b)
2. Class Templates: Generic classes create panna. template <class T> class Stack

Syntax:
template <class T> illa template <typename T>
T na template parameter. int, float, char, string edhu venalum varalam.

Explanation - Tamil la:
1. En Templates: add(int, int), add(float, float), add(double, double) nu 3 function ezhudura maari illa. Oru template potta podhum, compiler 3 um create pannidum.
2. Function Template: Xerox machine maari. Original template onu, aana neenga int kudu, float kudu, double kudu - adha adha ku copy eduthu kudukum.
3. Class Template: Container maari. Stack<int> na integer stack, Stack<string> na string stack. Orey class, aana data type mattum maari maari.
4. Compile Time: Runtime la illa, compile pannum bodhe add<int> ku thani function, add<float> ku thani function create aagidum. So fast ah work aagum.
5. STL: vector<int>, list<string> lam templates dhan. Namma use panra STL full ah templates la dhan irukku.

#include <string>

template <typename T>
T findMax(T a, T b) {
    return (a > b) ? a : b;
}

template <class T1, class T2>
void display(T1 x, T2 y) {
    cout << "First: " << x << " | Second: " << y << endl;
}

// --- Class Template ---
template <class T>
class Calculator {
private:
    T num1, num2;
public:
    Calculator(T n1, T n2) {
        num1 = n1;
        num2 = n2;
    }

    T add() {
        return num1 + num2;
    }

    T subtract() {
        return num1 - num2;
    }

    T multiply() {
        return num1 * num2;
    }

    void showResult() {
        cout << "Numbers: " << num1 << ", " << num2 << endl;
        cout << "Add: " << add() << endl;
        cout << "Subtract: " << subtract() << endl;
        cout << "Multiply: " << multiply() << endl;
    }
};

// --- Class Template with Multiple Parameters ---
template <class T1, class T2>
class Pair {
private:
    T1 first;
    T2 second;
public:
    Pair(T1 f, T2 s) : first(f), second(s) {}

    void show() {
        cout << "Pair: (" << first << ", " << second << ")" << endl;
    }
};

int main() {
    cout << "--- Function Template ---" << endl;
    cout << "Max of 10, 20: " << findMax(10, 20) << endl;
    cout << "Max of 5.5, 3.2: " << findMax(5.5, 3.2) << endl;
    cout << "Max of 'a', 'z': " << findMax('a', 'z') << endl;
    cout << "Max of \"Apple\", \"Orange\": " << findMax(string("Apple"), string("Orange")) << endl;

    display(100, 3.14);
    display("Age", 25);

    cout << "\n--- Class Template: int ---" << endl;
    Calculator<int> intCalc(10, 5);
    intCalc.showResult();

    cout << "\n--- Class Template: float ---" << endl;
    Calculator<float> floatCalc(10.5, 3.2);
    floatCalc.showResult();

    cout << "\n--- Class Template: Multiple Types ---" << endl;
    Pair<int, string> p1(1, "One");
    Pair<string, double> p2("PI", 3.14159);
    p1.show();
    p2.show();

    return 0;
}

1. getline(cin, str) - Full line ah space oda input edukkum.
2. resize(n) - String length ah n ku maathum. Kammi aan truncate aagum, adhigam aan default char fill aagum.
3. resize(n, ch) - Size perusa aan 'ch' character ah fill pannum.
4. clear() - String ah empty aakum. str = "" maari.
5. swap(str2) - Rendu strings ah content ah maathum.
6. shrink_to_fit() - Extra capacity ah release pannum. Capacity = size aagidum.
7. reserve(n) - Minimum n characters ku capacity reserve pannum.
8. stoi(), stol(), stof(), stod() - String ah int, long, float, double ku convert pannum. C++11 la vandhuchu.
9. to_string() - Number ah string ku convert pannum.
10. begin(), end() - Iterator kudukum. Loops ku use aagum.

2. Example Code - Clean Code

#include <iostream>
#include <string>
using namespace std;

int main() {
    cout << "--- getline() Example ---" << endl;
    string name;
    cout << "Enter full name: ";
    getline(cin, name);
    cout << "Name: " << name << endl;

    cout << "\n--- resize() Example ---" << endl;
    string s1 = "Hello World";
    cout << "Original: " << s1 << " | Length: " << s1.length() << endl;

    s1.resize(5);
    cout << "After resize(5): " << s1 << " | Length: " << s1.length() << endl;

    s1.resize(10, '*');
    cout << "After resize(10,'*'): " << s1 << " | Length: " << s1.length() << endl;

    cout << "\n--- clear() and empty() ---" << endl;
    string s2 = "Not Empty";
    cout << "Before clear: " << s2 << " | Empty? " << s2.empty() << endl;
    s2.clear();
    cout << "After clear: \"" << s2 << "\" | Empty? " << s2.empty() << endl;

    cout << "\n--- swap() Example ---" << endl;
    string a = "First";
    string b = "Second";
    cout << "Before swap - a: " << a << " | b: " << b << endl;
    a.swap(b);
    cout << "After swap - a: " << a << " | b: " << b << endl;

    cout << "\n--- reserve() and shrink_to_fit() ---" << endl;
    string s3;
    cout << "Initial capacity: " << s3.capacity() << endl;

    s3.reserve(100);
    cout << "After reserve(100): " << s3.capacity() << endl;

    s3 = "Small";
    cout << "After assigning Small - Size: " << s3.size()
         << " Capacity: " << s3.capacity() << endl;

    s3.shrink_to_fit();
    cout << "After shrink_to_fit - Size: " << s3.size()
         << " Capacity: " << s3.capacity() << endl;

    cout << "\n--- String to Number Conversion ---" << endl;
    string num1 = "123";
    string num2 = "45.67";
    string num3 = "99999";

    int i = stoi(num1);
    float f = stof(num2);
    long l = stol(num3);
    double d = stod(num2);

    cout << "stoi(\"123\"): " << i << " + 10 = " << i + 10 << endl;
    cout << "stof(\"45.67\"): " << f << endl;
    cout << "stol(\"99999\"): " << l << endl;
    cout << "stod(\"45.67\"): " << d << endl;

    cout << "\n--- Number to String Conversion ---" << endl;
    int x = 100;
    float y = 3.14;
    string str1 = to_string(x);
    string str2 = to_string(y);

    cout << "to_string(100): " << str1 << endl;
    cout << "to_string(3.14): " << str2 << endl;
    cout << "Concatenated: " << str1 + str2 << endl;

    cout << "\n--- Iterators begin() end() ---" << endl;
    string s4 = "ITERATE";
    cout << "Using iterator: ";
    for(auto it = s4.begin(); it!= s4.end(); it++) {
        cout << *it << " ";
    }
    cout << endl;

    cout << "Using range-based for: ";
    for(char c : s4) {
        cout << c << "-";
    }
    cout << endl;

    return 0;
}

UNIT I - Introduction to C++

1. Historical Background and Evolution

C++ is a massive, multi-paradigm, general-purpose programming language that was conceptualized and developed by Bjarne Stroustrup in the year 1979 at AT&T Bell Laboratories in Murray Hill, New Jersey. Initially, the language was named "C with Classes" because it was designed to be an extension of the highly popular C language, incorporating the object-oriented concepts from the Simula67 language without compromising the efficiency and speed of C.

In 1983, the language was officially renamed C++. The term "++" is the increment operator in C, symbolically representing that C++ is an advanced, incremented version of C. The first commercial release of C++ occurred in 1985. It was standardized by the ISO (International Organization for Standardization) in 1998 (C++98), followed by multiple modernized versions like C++11, C++14, C++17, C++20, and C++23, making it one of the most powerful and heavily utilized languages in engineering, game development, and systems programming today.

Tanglish Explanation (History): C++ language-ah 1979-la 'Bjarne Stroustrup' nu solra avar thaan uruvakkunaar. Starting-la idhuku "C with Classes" nu per vachangha. Enna na, C language-oda fast execution kooda Simula67 language-oda Object-Oriented concepts-ah mix panradhukaaga idhu ready aachis. Apram 1983-la idhuku 'C++' nu per maathunangha. C language-la nammalku therinja '++' (Increment Operator) vechu, idhu C-oda adutha level periya version-nu kaatadhuku thaan intha name vachangha.

2. Core Architectural Features of C++

C++ is regarded as a Middle-Level Language because it seamlessly bridges the gap between low-level (hardware-centric) programming and high-level (abstract application-centric) programming. Below are the key characteristics that define C++:

  • Robust and Fast: Being a compiled language, C++ translates source code directly into machine-executable binary code, leading to incredibly fast execution times, making it ideal for systems software.
  • Portability: C++ is platform-independent in terms of concept (ANSI standard), meaning code written on one machine can be compiled and executed on another machine with minimal modifications.
  • Rich Library Support: It offers an extensive Standard Template Library (STL) containing pre-built data structures like vectors, lists, maps, and rich algorithms that minimize manual coding efforts.
  • Memory Management: Unlike managed languages like Java, C++ gives programmers direct control over memory allocation and deallocation using the new and delete operators, allowing explicit structural optimization.
Tanglish Explanation (Features): C++ oru 'Middle-Level Language'. Ithanala computer hardware-ah direct-aa control panna mudiyum (Low-level), adhey neram periya high-level applications-ayum ready panna mudiyum. Idhu romba fast-aa run aagum, enna na idhu oru compiled language. Idhula memory-ah direct-aa 'new' matrum 'delete' operators moolama programmer-ey manage pannikalam.

3. Object-Oriented Programming (OOP) Paradigms

The core power of C++ lies in its support for Object-Oriented Programming. A 10-mark architectural analysis requires understanding its four foundational pillars:

  1. Encapsulation: The process of binding data variables and the functions that manipulate them into a single comprehensive blueprint unit called a Class. This prevents unauthorized direct data access.
  2. Data Abstraction: Hiding complex internal implementation complexities and showing only the essential interfaces to the outside world. This is achieved using access specifiers like public, private, and protected.
  3. Inheritance: The mechanism by which a new class (Derived Class) acquires the properties, states, and behaviors of an existing class (Base Class), maximizing code reusability.
  4. Polymorphism: The ability of data or functions to process in more than one form. It manifests as Compile-time polymorphism (Function overloading, Operator overloading) and Runtime polymorphism (Virtual functions).
Tanglish Explanation (OOP Pillars): C++ oda main balamey intha OOP concepts thaan. Idhula motham 4 mukkiya pillars irukku: 1) Encapsulation - Data-vaiyum function-aiyum 'Class' nu solra ore safety box-kulla potu lock panradhu. 2) Abstraction - Devaiyillatha internal logic-ah maraichu, main features-ah mattum veliya kaatradhu. 3) Inheritance - Oru class-oda properties-ah innoru puthu class reuse pannikradhu. 4) Polymorphism - Oru function oru vagaiuku mela vera vera formats-la perform panradhu.

UNIT I - Key Concepts of Object-Oriented Programming (OOP)

1. Paradigm Evolution & Core Philosophy

Object-Oriented Programming (OOP) is an advanced programming paradigm that models real-world entities using software computational constructs. Historically, traditional structural or procedural programming paradigms (like C, Pascal) focused strictly on writing algorithmic procedures and data processing steps. In procedural architecture, data was treated as a secondary entity and moved freely across global functional boundaries, which exposed it to accidental mathematical corruptions and major structural vulnerabilities.

OOP fundamentally re-engineers this approach by shifting the developmental focus entirely onto Data Security and data structure states. Instead of breaking software down into functional steps, OOP breaks systems down into individual self-sustaining components called Objects. It binds data variables closely to their corresponding operations, completely isolating internal computational logic from outside interruptions.

Tanglish Explanation (OOP Philosophy): Normal procedural programming (C language madhiri) structure-la logic matrum functions-ku thaan mukkiyathuvam thandhangha, data-ku strong security illa. Aana OOP paradigm-la data security-ku thaan primary importance tharaangha. Software-ah function step-by-step-aa pirikaama, real-world components aana "Objects" ah base panni design pannuvaangha. Data-vum adhai corporate panra functions-um ore idathula lock aiyirukum.

2. In-Depth Technical Analysis of OOP Elements

To analyze Object-Oriented software architectures, university curricula evaluate the following core architectural elements:

A. Objects

An Object is the fundamental runtime entity in an object-oriented paradigm. It represents an instance of a structural class design template. Physically, an object occupies computational memory space inside the computer's RAM. It holds data properties (Attributes) and executes functional logic operations (Behaviors). For example, if 'Car' is a class template, then a specific physical vehicle with a license plate is an object possessing its own localized state values.

B. Classes

A Class is a user-defined logical data type block that functions as a structural blueprint or master template from which individual runtime objects are manufactured. A class defines the precise types of variables and functional interfaces that its objects will possess. It does not consume physical hardware memory space during compilation; it merely dictates the organizational metadata rules for memory allocations.

Tanglish Explanation (Objects & Classes): Class-ngradhu oru structural template illa-na user-defined blueprint madhiri. Idhu compiler computational level-la memory space-ah eduthukaadhu. Oru building-oda engineer structural plan sheet madhiri. Object-ngradhu intha class template-ah vechu RAM memory-la run aagura unmaiyana real instance entity. Concept-padi Plan sheet-ngradhu Class, adhai vechu kattradha building dhaan Object.

C. Encapsulation & Data Hiding

Encapsulation is the physical process of bundling structural data variables and methods together within a single unified class box. Within this encapsulated zone, data fields are kept private (hidden) using the private: access specifier. This technique is formally called Data Hiding. The data cannot be modified directly by external global code loops; it can only be safely manipulated through public access interfaces (Getters and Setters).

D. Abstraction

Abstraction is the systemic philosophy of representing only the essential operational features of a component without exposing its deep internal implementation background or complex algorithm routines. For instance, when a driver pushes a car's brake pedal, they experience the conceptual abstraction of slowing down, completely unconcerned with the complex hydraulic fluid mechanics happening inside the mechanical engine assembly.

Tanglish Explanation (Encapsulation & Abstraction): Data variables-ayum methods-ayum ore class kulla wrapping panni lock panra process-ku per Encapsulation. Idhula variables-ah private-aa maraikradhuku per Data Hiding. Abstraction-ngradhu complex processing-ah pinnaadi maraichutu essential features-ah mattum veliya kaatradhu. Mobile-la power button-ah amukki screen on panra abstraction therinja podhum, ulla motherboard-la IC chips eppadi electrical current anupudhunu user therinjuka devaiyilla.

E. Inheritance (Extensibility Architecture)

Inheritance is the structural framework that allows a brand new class block (Derived/Child Class) to inherit, adapt, and reuse the existing methods and member fields of an older class layout (Base/Parent Class). This eliminates redundant coding efforts, supports modular software scaling, and introduces precise hierarchy tracking into complex applications.

F. Polymorphism

Polymorphism translates from Greek as "the ability to take multiple distinct operational forms". It allows the same functional interface call signature to trigger entirely different underlying execution paths based on context. In C++, compile-time polymorphism handles static function and operator overloading, while runtime polymorphism uses dynamic virtual pointers to decide function executions on the fly.

Tanglish Explanation (Inheritance & Polymorphism): Oru pazhaya parent class-oda code structures-ah puthu child class ezhuthama reuse panradhuku per Inheritance. Ithanala line coding time save aagum. Polymorphism-na orey name-la irukura interface routine vera vera dynamic forms-la work aagradhu. Static overloading matrum runtime virtual mechanisms-ah vechu C++ la compiler itha execute pannudhu.

3. Production-Grade OOP C++ Demonstration

The following structural program demonstrates the clean operational syntax of Class construction, private encapsulation, object instantiation, and public functional manipulation, completely free from comment lines:

#include <iostream>
#include <string>

using namespace std;

class AcademicStudent {
private:
    string studentName;
    int internalMarks;

public:
    void setupData(string name, int marks) {
        studentName = name;
        if (marks >= 0 && marks <= 100) {
            internalMarks = marks;
        } else {
            internalMarks = 0;
        }
    }

    void displayReport() {
        cout << "Student System Profile: " << studentName << endl;
        cout << "Secured Internal Assessment Marks: " << internalMarks << "/100" << endl;
    }
};

int main() {
    AcademicStudent studentObject1;
    AcademicStudent studentObject2;

    studentObject1.setupData("Nisha", 98);
    studentObject2.setupData("Sathish", 85);

    cout << "--- University OOP Execution Terminal ---" << endl;
    studentObject1.displayReport();
    cout << "----------------------------------------" << endl;
    studentObject2.displayReport();

    return 0;
}

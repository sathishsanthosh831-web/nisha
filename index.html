UNIT I - Introduction to C++

C++ is a general-purpose programming language created by Bjarne Stroustrup in 1985 as an extension of the C language. It was designed to add object-oriented features to C while maintaining C's efficiency and flexibility.

Key Features of C++

  1. Object-Oriented Programming: C++ supports OOP concepts like classes, objects, inheritance, polymorphism, encapsulation, and abstraction. This helps model real-world problems using objects.
  2. Middle-Level Language: C++ combines features of both high-level and low-level languages. You can do system programming like C and also build large applications with OOP.
  3. Compiled Language: C++ code is compiled into machine code, which makes programs run very fast compared to interpreted languages.
  4. Rich Library Support: C++ provides Standard Template Library (STL) which includes built-in data structures and algorithms like vectors, stacks, queues, sorting, etc.
  5. Platform Dependent: C++ programs need to be recompiled for different operating systems, unlike Java which is platform independent.

Why C++ was Introduced

C language had limitations for large software development:

  1. No data security because of global variables
  2. No code reusability
  3. Difficult to maintain large codebases

C++ solved these by introducing classes and objects, which bind data and functions together. This is called encapsulation.

Key Concepts of Object-Oriented Programming (OOP) - Detailed

Object-Oriented Programming is a programming style based on the concept of "objects". An object contains both data and functions that operate on that data. The 4 main pillars of OOP make C++ powerful for large software development.

1. Class and Object - The Foundation

Before understanding the 4 pillars, you must know what is a class and object.

Class: A class is a blueprint or template. It does not occupy memory. Example: The design plan of a house.

Object: An object is an instance of a class. It occupies memory. Example: The actual house built using the design plan.

class Student {        // Class - Blueprint
    public:
        string name;
        int rollNo;
        void display() {
            cout << name << " " << rollNo;
        }
};

int main() {
    Student s1;        // Object - Real entity
    s1.name = "Ravi";
    s1.rollNo = 101;
    s1.display();
}

2. Encapsulation - Data Hiding

Definition: Encapsulation is the process of binding data members and member functions together into a single unit called a class. It also restricts direct access to some of the object's components.

Real Life Example: A company has different departments like Accounts, Sales, HR. The Accounts department data is hidden from the Sales department. Only authorized functions can access it.

How C++ Achieves It: Using access specifiers: private, public, protected. We keep data as private and provide public functions to access it. These functions are called getters and setters.

class BankAccount {
    private:
        double balance;  // Data is hidden - cannot access directly
        
    public:
        void setBalance(double b) {  // Setter function
            if(b >= 0) balance = b;
        }
        double getBalance() {        // Getter function
            return balance;
        }
};

Main Benefits:
1. Data Security: Private data cannot be changed accidentally from outside.
2. Control: You can add validation logic inside setter functions.
3. Flexibility: You can change internal code without affecting other parts of the program.

3. Inheritance - Code Reusability

Definition: Inheritance is the process where one class acquires the properties and behaviors of another class. The class that inherits is called "Derived Class" or "Child Class". The class from which it inherits is called "Base Class" or "Parent Class".

Real Life Example: A "Car" is a type of "Vehicle". Vehicle has common properties like speed, color. Car inherits these properties and also has its own properties like number of doors, AC.

Types of Inheritance in C++: Single, Multiple, Multilevel, Hierarchical, Hybrid.

class Vehicle {          // Base Class
    public:
        int speed;
        void run() {
            cout << "Vehicle is running";
        }
};

class Car : public Vehicle {  // Derived Class - inherits Vehicle
    public:
        int doors;
        void honk() {
            cout << "Car is honking";
        }
};

int main() {
    Car c1;
    c1.speed = 100;  // Inherited from Vehicle
    c1.doors = 4;    // Own property
    c1.run();        // Inherited function
    c1.honk();       // Own function
}

Main Benefits:
1. Code Reusability: Write common code once in base class and use it many times.
2. Saves Time: No need to rewrite same logic.
3. Establishes Relationship: Creates a natural hierarchy like Animal -> Dog -> Puppy.

4. Polymorphism - One Name Many Forms

Definition: Polymorphism means "many forms". It allows you to perform a single action in different ways. The word is derived from Greek: "Poly" means many and "morph" means forms.

Real Life Example: The word "cut" has different meanings. You cut paper, cut vegetables, cut a movie scene. The action depends on the object.

Two Types in C++:
A. Compile Time Polymorphism: Decided during compilation. Achieved by Function Overloading and Operator Overloading.
B. Run Time Polymorphism: Decided during program execution. Achieved by Function Overriding using virtual functions.

// Function Overloading - Compile Time
class Print {
    public:
        void show(int i) {
            cout << "Integer: " << i;
        }
        void show(string s) {
            cout << "String: " << s;
        }
};

// Function Overriding - Run Time
class Animal {
    public:
        virtual void sound() {
            cout << "Animal makes sound";
        }
};

class Dog : public Animal {
    public:
        void sound() override {
            cout << "Dog barks";
        }
};

Main Benefits:
1. Flexibility: Same function name can be used for different data types.
2. Maintainability: Code becomes cleaner and easier to extend.
3. Interface Consistency: You can call sound() on any Animal object.

5. Abstraction - Hiding Complexity

Definition: Abstraction means displaying only essential information to the user and hiding the complex implementation details. It focuses on "what" an object does instead of "how" it does it.

Real Life Example: When you use a TV remote, you just press the power button. You don't need to know the circuit board, infrared signal, or internal wiring. That complexity is hidden from you.

How C++ Achieves It: Using Abstract Classes and Interfaces. An abstract class has at least one pure virtual function. You cannot create an object of an abstract class.

class Shape {  // Abstract class
    public:
        virtual void draw() = 0; // Pure virtual function
};

class Circle : public Shape {
    public:
        void draw() override {
            cout << "Drawing Circle"; // User sees only this
            // Complex drawing logic is hidden here
        }
};

int main() {
    // Shape s;  // Error: Cannot create object of abstract class
    Circle c;
    c.draw();    // User just calls draw(), complexity is hidden
}

Main Benefits:
1. Reduces Complexity: User deals with a simple interface.
2. Increases Security: Internal implementation cannot be misused.
3. Improves Maintainability: You can change internal code without changing user code.
